# OpenCodugin

![Build](https://github.com/hunterxxn/OpenCodugin/workflows/Build/badge.svg)

Integrate [opencode](https://github.com/anomalyco/opencode) CLI into IntelliJ IDEA, providing full terminal mouse support and IDE context sharing.

## Introduction

Due to years of development habits, I still prefer using IDEA as my daily development environment. In the current AI wave, the earliest integration form was IDE + CLI tools.

In recent years, many excellent Agents have emerged. Among them, OpenCode is the Agent tool I've been using from the very beginning and continue to use. Open source, compatible, with a great ecosystem — OpenCode's advantages are self-evident.

However, using it directly in the IDEA Terminal revealed many issues, including but not limited to: mouse wheel not scrolling chat history accurately, the right scrollbar jumping with mouse movement, shortcut key conflicts (sometimes wanting to copy but accidentally closing the process)...

Additionally, I wanted to add features like the ability to generate file references in the terminal through code shortcuts, since OpenCode supports @ file references but lacks convenient code-based insertion.

This inspired me to create an IDEA plugin using AI. Before writing it, I also looked for existing plugins — there was one for OpenCode, but the user experience wasn't quite what I envisioned, so I decided to build my own.

I've been using it privately — essentially wrapping OpenCode with some auxiliary features — and didn't think it was worth releasing. But recently, Mimo Code, developed based on OpenCode, was also released. I realized I could integrate all the TUI tools I commonly use, making this tool much more versatile.

Currently supports OpenCode, MimoCode, and Reasonix (version 1.x can be launched directly, version 0.x can be launched via Custom input `npx reasonix code`, which was the previously recommended way to start it).

I originally didn't want to open-source this — not because I'm protective of it. First, the code wasn't written by me; it was all generated by AI. I just pressed Enter to execute. I don't know Kotlin myself and can only plan with a Java mindset, and my code review isn't rigorous.

Moreover, people don't really read code to learn these days, so open-sourcing seems to have limited value. But publishing to the marketplace requires filling in an address, so using a GitHub address after open-sourcing works better.

In any case, I hope this helps everyone. It's a small vibe-coding creation. All code was generated by DeepSeek V4Pro and Mimo V2.5Pro, with possibly one use of GLM 5.1. Domestic models are actually very capable. After all, developing an IDEA plugin has far fewer resources than Spring, and solving the problems encountered was no small feat.

I hope domestic model vendors keep pushing forward, surpass those closed models, and help us achieve token freedom soon, haha (●'◡'●)

## Features

- **Native PTY Terminal** — Full terminal emulation based on Jediterm, with mouse event reporting and scrolling
- **Multi-session Management** — Open independent CLI sessions in tabs
- **Bottom Tool Window** — Pinned to the bottom of the IDE, always accessible
- **IDE Context Sharing** — CLI automatically uses the project root directory as the working path
- **File Reference Injection** — Right-click a file to inject a ` @path#LlineNumber ` reference into the active terminal session
- **Multi CLI Support** — Supports OpenCode, Mimo, and custom commands

<kbd>Settings/Preferences</kbd> > <kbd>Plugins</kbd> > <kbd>Marketplace</kbd> > Search **OpenCodugin** > <kbd>Install</kbd>

Download the latest version from [GitHub Releases](https://github.com/hunterxxn/OpenCodugin/releases/latest), then:

<kbd>Settings/Preferences</kbd> > <kbd>Plugins</kbd> > <kbd>⚙️</kbd> > <kbd>Install plugin from disk...</kbd>

## Prerequisites

- IntelliJ IDEA 2024.2 or higher
- At least one of the following CLI tools installed:
  - [opencode](https://github.com/anomalyco/opencode)
  - [mimo](https://github.com/anthropics/mimo)
  - or any other command-line tool

## Usage

### Getting Started

Open the **OpenCodugin** tool window at the bottom of the IDE. A CLI selection menu will appear automatically:

| Option | Description |
|--------|-------------|
| OpenCode | Use the opencode CLI installed on the system |
| Mimo | Use the mimo CLI installed on the system |
| Custom... | Enter a custom command to start |

A session is created automatically after selection and the terminal starts.

### Toolbar Buttons

| Button | Function |
|--------|----------|
| New Session | Create a new session tab with the CLI selection menu |
| Stop | Terminate the current session |
| Check for Updates | Check if a new plugin version is available |

### Custom Commands

After selecting **Custom...**, enter any command in the input dialog. The following formats are supported:

#### Single Command

```
opencode
mimo
node
python
```

Enter the command name directly; the plugin will automatically search the system PATH and resolve it to a full path (including `.cmd`, `.exe`, and other extensions).

#### Command with Arguments

```
npx reasonix code
node server.js --port 3000
python -m http.server 8080
```

The command and arguments are separated by spaces. The plugin will:
1. Resolve the first word as the executable path (e.g., `npx` → `C:\Users\...\npm\npx.cmd`)
2. Pass the rest as arguments

#### Full Path

```
C:\Tools\my-cli.exe
/usr/local/bin/custom-tool
D:\dev_program\some-tool\bin\tool.cmd
```

When the input contains `\` or `/`, it is treated as a full path and no splitting or resolution is performed.

### File Reference Injection

Right-click a file in the following locations and select **Add to OpenCode**:

| Location | Behavior |
|----------|----------|
| Project View (File Tree) | Inserts the file path relative to the project root |
| Editor Tab | Inserts the file path relative to the project root |
| Editor Content Area | Inserts the file path; appends line number range when text is selected |

The reference is inserted into the terminal as editable text. You can add context before pressing Enter to send. If there is no active session, the reference is copied to the clipboard.

Multi-select is supported: select multiple files, right-click, and all references are inserted at once, separated by spaces.
